# Memory

What your experts remember, stored entirely on this device, in three layers:

| Layer | Scope |
|---|---|
| Shared | Facts about you — visible to all experts |
| Role | What one specific expert has learned |
| Collab | Learned across expert hand-offs |

## Manage memories

Settings → Memory:

- Filter by Expert and Layer; edit or delete any entry; "Open source conversation" jumps to where a memory was formed.
- The **Self-learning** toggle controls whether experts save useful context from your conversations automatically.

## Memory Live

Type `/memory` in the composer to open Memory Live — a 3D cloud of everything remembered. Node size tracks content, brightness tracks how recently it was recalled; recalls flash live while experts work. Click a node to open its source conversation.

## Expert note-taking

Agents also keep their own working notes as they go — you'll see `remember`, `recall_memory` and `forget` tool cards in chat. They likewise maintain per-project maps (`remember_project_map`): a persistent summary of what they learned about a codebase, keyed to the working folder, so the next session starts oriented.

Ask directly: "remember that our staging URL is …" or "forget what you saved about X".

## Profile

Settings → Profile is the explicit layer you write yourself — Display Name, Role / Occupation, Primary Stack / Domain, Preferred Tone, Preferred Reply Language, Timezone, About Me. It is appended to every request so every expert knows who they're working with.

## Notes

- Deleting a role removes that role's layer; the Shared layer remains.
- Nothing in any layer leaves this device.
